PDF Plot Generation from TLO Manifest

Problem

At present there is limited oversight to train loading. Data is currently collected for every wagon loaded but it is rarely used and never visualised. This causes loading inefficiencies to occur.

Solution

Create a python script to interpret the manifest collected from SCADA and generate a PDF with plots of relevant train loading data.

Have the pdf automatically emailed out to stakeholders after the completion of every train.

The PDF enhances oversight and helps the team identify and address operational issues. The visualisations make it easy to identify abnormal loading scenarios and detect excessive manual operations.

Creation from PlantSCADA

After a train is completed, an event is triggered within PlantSCADA which executes a cicode function. This creates a .txt file with every wagon's loading data.

Files Needed for Processing and Emailing

At the end of the cicode function a yython script packaged as an executable is called. This processes the latest txt file and outputs the PDF plots. Next, a batch file for emailing the PDF manifest to recipients is triggered.

Next, a batch file for emailing the PDF manifest is triggered which calles blat.exe, a SMTP mailer.

All recipients of the pdf plot are listed in a txt file, this allows the recipient list to be updated easily.

Example PDF

Performance summary with time/weight target metrics followed by 7 loading plots.

Heavy use of multi-axis plots to correlate dependant loading variables to inform decision making during abnormal situations
